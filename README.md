# walk-sensor-camera（カメラ連動 歩行検知システム）

カメラで人の入室を検知し、そのゾーンに入った人の歩行だけを検知して、離れた場所のPCで足音を鳴らすシステムです。イベントのブース展示などで、「そのブースに入ってきた人」に反応して足音を鳴らすことを想定しています。

## 背景と目的

歩行検知システム（[walk-sensor](../walk-sensor)）を、イベントのブースで使うために発展させたものです。

イベントでは、参加者に事前にURLを開いてもらいます。しかし、全員が常に歩行を送信すると、ブースの外にいる人の歩行にも反応してしまいます。そこで、ブースの入口にカメラを置き、人が入ってきたことを検知したときだけ、歩行の送信を有効にする仕組みにしました。

## システム構成

3つの機器が Firebase Realtime Database を介して連携します。

| 機器 | 役割 | 使うもの |
|---|---|---|
| Raspberry Pi + カメラ | 人の入室を検知し `zone` に書き込む | Python / MediaPipe / picamera2 |
| スマートフォン | 歩行を検知し、ゾーン内のときだけ `step` に送信 | `index.html`（ブラウザ） |
| PC | `step` を受信して足音を鳴らす・歩数を表示 | `pc.html`（ブラウザ） |

### データの流れ
[ラズパイ カメラ] --人検知--> Firebase(zone)
|
v
[スマホ] --zoneを確認--> ゾーン内なら--> Firebase(step)
|
v
[PC] --> 足音・カウント

## 各ファイルの役割

- `index.html`：スマートフォン用。歩行を検知し、Firebaseの `zone` が「人がいる」ときだけ `step` に送信する。ゾーン外では自分の歩数をカウントするだけで送信しない。
- `pc.html`：PC用。`step` を受信して足音を鳴らし、歩数を表示する。`zone` の状態（人がいるか）も表示する。
- ラズパイ側のプログラム（別リポジトリ [camera-audio](../camera-audio) の `src/zone_detect.py`）：カメラで人を検知し、状態が変わったときに Firebase の `zone` に書き込む。

## 使い方

### 1. ラズパイ（カメラ）を起動
ラズパイで人検知プログラムを実行します。カメラの前に人がいるかどうかを検知し、Firebaseに送信します。

```bash
cd camera-audio
source .venv/bin/activate
python3 src/zone_detect.py
```

### 2. PC側を準備
PCのブラウザで `pc.html` を開きます。音を鳴らす場合は「音を有効にする」を押します。

### 3. スマホ側を準備
スマートフォンのブラウザで `index.html` を開き、「スタート」を押します（iOSはモーション許可が必要。消音モードは解除）。

### 4. 動作
- カメラの前に誰もいないとき：スマホは歩数をカウントするだけで、PCは反応しない（ゾーン外）。
- カメラが人を検知したとき：スマホが「ゾーン内」になり、歩行がPCに送信され、足音が鳴る（ゾーン内）。

## 歩行検知の仕組み

加速度センサーの値から重力を除いた「動きの加速度」を計算し、着地のたびに現れる山（ピーク）を検出します。単発の動きを誤検知しないよう、一定のリズムで山が続いたときだけ「歩行中」と判定します。

## 人検知の仕組み

Raspberry Pi のカメラ映像を MediaPipe Pose で解析し、人の姿勢（骨格）が検出できたら「人がいる」と判定します。状態が変わったときだけ Firebase に書き込むことで、無駄な通信を抑えています。

## 技術

- HTML / JavaScript（DeviceMotion API）
- Python / MediaPipe / picamera2（Raspberry Pi での人検知）
- Firebase Realtime Database（3機器間のリアルタイム連携）
- GitHub Pages（HTTPS配信）

## 既知の制約・今後の課題

- 複数人が同時にゾーンに入った場合、URLを開いている全員のスマホが送信状態になる。現状は「一人ずつ入る」運用を想定している。
- 足音は現在ビープ音。実際の足音ファイルへの差し替えは今後の課題。
